Init 你的螃蟹搭子

2026 · April Claude Code · CLAUDE.md · /init

螃蟹搭子挺好用的，但有个问题——它不知道你的游戏规则。

你喜欢 tab 还是空格？能不能直接 push 代码？爱听长篇大论还是直给答案？这些它都不知道。

而在复杂项目里，没有提前准备好的规则说明书，我们就需要反复的向 cc 介绍背景、提示词、规范，还会常常面临失忆问题。

CLAUDE.md 就是一份规则说明书。

CLAUDE.md 是什么

简单说，就是你和 AI 的项目说明书。它放在项目目录里，cc 每次进来都会先看一遍。

内容包括四块：

  • →项目概述
  • →技术栈信息
  • →代码风格指南
  • →开发命令（构建、测试、运行）

两个位置，优先级不同

CLAUDE.md 有多个位置可以放：

~/.claude/CLAUDE.md — 全局基础配置，所有项目共享

项目目录/CLAUDE.md — 项目特定配置，优先级更高

如果两边有冲突的规则，项目级别优先。

举个例子：全局配置说"用中文交流"，但某个项目是英文团队的项目，项目目录下的 CLAUDE.md 说"用英文"，那 cc 就会用英文。

—— 全局设置是个 baseline，每个项目可以有自己的 override ——

一份合格的全局 CLAUDE.md 需要什么

别写太长，200 行以内最优，重点是这四块：

  • →沟通方式
  • →工作习惯
  • →编码偏好
  • →边界与安全

给你一份可直接用的模版

复制下面的内容，根据你的情况改一改，就能用了：

# ~/.claude/CLAUDE.md

## 沟通方式

- 中文沟通
- 需求模糊时先提问澄清，不要猜
- 回复简洁直接，不要过度解释
- 发现更优路径时主动建议，不只按指令做

## 工作习惯

- 先有方案再写代码，需批准后再动手
- 被纠正后总结原因并写入 Memory
- 修改超过 3 个文件时，先拆成小任务

## 编码偏好

- 修改现有代码时，保持原有风格
- 优先使用简单直接的方案
- 不要留下调试用的 console.log
- 变量命名清晰，不要用单字母或是缩写
- 不要为假想需求预留拓展点

## 边界与安全

- 不要自动提交或推送代码
- 不要未经确认添加新依赖
- 不要重构未涉及的代码
- 不要在代码中硬编码密钥和密码

复杂项目，用 /init 更省心

平常进项目是这样的：

cd your_project
claude

简单项目够用，复杂项目建议用 /init。

这个命令会帮你做三件事：

  1. 创建或更新 CLAUDE.md 文件
  2. 自动分析项目结构，检测包管理器、构建工具、测试框架、主要语言和框架
  3. 生成项目配置，让后续对话更好地理解上下文

怎么用？进入项目目录后，在 Claude Code 会话里直接输入：

/init

cc 会扫描你的项目，然后问你想在 CLAUDE.md 里补充哪些规则。第一次配置可能花 2-3 分钟，后面每次对话它都能按这个来。

好的搭子从约定开始。

花 5 分钟写清楚说明书，后面能省下无数个来回
